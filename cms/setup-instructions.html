Setup Instructions

Running the CMS Locally

To run the CMS locally, you need to serve the files through a web server to avoid CORS issues. Here are a few options:

Option 1: Use a Simple HTTP Server

If you have Python installed, you can use its built-in HTTP server:

# Navigate to your website root directory
cd "C:\Users\Passionet\Desktop\wehustle.it.com\WH Site"

# Python 3.x
python -m http.server 8000

# Or Python 2.x
python -m SimpleHTTPServer 8000

Then open http://localhost:8000/cms/admin.html in your browser.

Option 2: Use Node.js http-server

If you have Node.js installed, you can use http-server:

# Install http-server globally (if not already installed)
npm install -g http-server

# Navigate to your website root directory
cd "C:\Users\Passionet\Desktop\wehustle.it.com\WH Site"

# Start the server
http-server

Then open http://localhost:8080/cms/admin.html in your browser.

Option 3: Use VS Code Live Server Extension

If you're using Visual Studio Code, you can install the "Live Server" extension:

  1. Install the "Live Server" extension from the VS Code marketplace
  2. Right-click on index.html in the file explorer
  3. Select "Open with Live Server"
  4. Navigate to the CMS admin page in your browser

Using the CMS

Once you have the server running:

  1. Access the admin dashboard at /cms/admin.html
  2. Login with the default credentials:
    • Username: admin
    • Password: admin
  3. Create and manage your blog content
  4. View your blog at /blog.html

Important Notes

  • This CMS uses localStorage for data persistence in this local development setup
  • In a production environment, you would need a proper backend server
  • The TinyMCE editor is using a valid API key with premium features enabled
  • This key is for development purposes only - for a production site, you should get your own API key
